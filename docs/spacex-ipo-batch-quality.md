## 和此前手动新增的差异

手动新增（如 Apple、Nvidia、Intel、Micron、中船特气）质量更高的地方：

- 供应链关系更具体，可以写到特定供应商、客户、产品/服务关系和上市代码。
- 业务分部和收入构成更完整，更适合做历史百分比图。
- 最新动态可以基于公司公告、新闻稿和行业消息做更细的影响判断。

批量流程更好的地方：

- 新增速度明显更快，适合先建立 watchlist 和基础页面。
- SEC 数值可追溯性更强，能稳定拿到 CIK、filings、年度趋势和 Q1 数据。
- 可批量发现质量问题，再集中制定规则。

## 本次发现并修正的质量问题

1. SEC `companyfacts` 的 `quarterly` 候选中，Q2/Q3 常出现年初至今口径。如果直接作为单季度收入或现金流，会高估单季数据。本次只使用年度趋势和明确的 Q1 单季度值。
2. ASTS 的收入序列在自动稿中不完整且偏旧，不适合放入收入趋势图。本次改用净亏损和总资产趋势，并在收入构成中标注未采用该序列。
3. Viasat 财年截至 3 月 31 日，不能按自然年解释。本次所有 VSAT 年度指标都标注为公司财年 FY2024/FY2025/FY2026。
4. SpaceX 仍是非上市公司，本次没有把 SpaceX 当作可查询公司页新增，而是作为 SpaceX / Starlink 主题锚点写入下游关系。
5. 批量新增没有自动声称某公司是另一公司的直接供应商或客户；只有在公开披露足够明确时，后续才应升级为具名上市公司关系。

## 后续优化建议

- 已在 `research_pipeline.py` 中增加季度口径过滤：优先使用带 `CYxxxxQx` 或 `CYxxxxQxI` frame 的单季度/期末值，剔除无 frame 的 YTD 候选。
- 为 `companyfacts` 年度数据增加排序和去重检查，避免把比较期误读为最新期。
- 新增 `qualityFlags` 输出：例如 `quarterly_ytd_risk`、`segment_revenue_unreviewed`、`revenue_series_incomplete`。
- 对特定关系使用二次来源确认：SEC 10-K、重大合同公告、公司 IR 新闻稿至少命中一个后，才写入具名供应商/客户。
- 在未来启用定时任务前，继续保持 draft-only 工作流，由人工 review 后再落库。
